 Программно-аппаратная система мониторинга окружающей среды на базе ESP8266

Репозиторий содержит полный исходный код, конфигурационные скрипты и схемы для автономной метеостанции с беспроводной передачей данных. Система построена на базе микроконтроллера ESP8266, осуществляет сбор телеметрии, кэширование данных через каналы Telegram, синхронизацию с СУБД PostgreSQL и предоставление интерфейса пользователю.

 Структура проекта
- /firmware — встроенное ПО микроконтроллера ESP8266 (C++, Arduino IDE).
- /server — серверная часть системы (Python):
  - bot.py — интерфейсный Telegram-бот (архитектура FSM, обработка команд, меню).
 	- esp_sync.py — асинхронный фоновый синхронизатор данных на базе Telethon.
  - graph.py — модуль визуализации данных и построения временных графиков (matplotlib).
  - report_utils.py — модуль аналитической обработки и экспорта отчетов в Excel (pandas).
- /server/database — конфигурационные скрипты СУБД PostgreSQL и шаблоны настроек среды.
- /hardware — принципиальная схема подключения периферии и датчиков.

 Архитектура и стек технологий

 Аппаратная часть (ПО измерительного модуля)
- Микроконтроллер: ESP8266.
- Сенсоры: Датчик климатических параметров BME280/BMP280, датчик освещенности MAX44009.
- Локальный вывод: Символьный дисплей LCD 16x2 по шине I2C.
- Интерфейсы: Локальное кнопочное физическое меню, встроенный локальный веб-сервер (HTML-страница с текущей телеметрией), отправка HTTPS-запросов к Telegram Bot API.

 Серверная часть и СУБД
- База данных: Реляционная СУБД PostgreSQL. В папке /server/database находится SQL-скрипт для инициализации структуры, содержащий правила каскадного удаления данных (ON DELETE CASCADE) и индексацию первичных/внешних ключей для ускорения транзакций временных рядов.
- Фоновый синхронизатор (esp_sync.py): Python, асинхронная библиотека Telethon (клиентская сессия). Выполняет разбор пула сообщений из кэш-группы Telegram на основе маркера last_message_id.
- Интерфейсный Telegram-бот (main_bot.py): Python, фреймворк python-telegram-bot. 
  - Реализован механизм конечных автоматов (FSM) для пошаговых сценариев (регистрация устройств).
  - Иерархическое меню с автоматическим удалением/редактированием сервисных сообщений чата.
  - Разделение прав доступа на базовую (пользовательскую) и административную роли.
  - Фоновый асинхронный контур уведомлений (превышение лимитов, потеря связи со станцией).

 Аналитический контур (Отчеты и графики)
- Табличные отчеты: Библиотеки pandas и openpyxl. Генерация документов Excel (листы «Датчики» и хронологическая таблица «Показания» с автоматическим исключением избыточных системных ID).
- Графический модуль: Библиотека matplotlib. Автоматическая временная агрегация данных (усреднение в 10 минут, час или сутки в зависимости от масштаба запроса) с цветовой маркерной индикацией особых событий (зеленый — ручная отправка, красный — выход за технологические границы).

 Инструкция по развертыванию

 1. Аппаратная прошивка
1. Откройте файл /firmware/firmware.ino в Arduino IDE.
2. Установите библиотеки Wire, LiquidCrystal_I2C, Adafruit_BME280, Adafruit_MAX44009.
3. Укажите параметры вашей Wi-Fi сети и токен технического Telegram-бота.
4. Прошейте плату ESP8266.

 2. Настройка базы данных
1. Разверните сервер PostgreSQL (локально или удаленно).
2. Выполните SQL-скрипт инициализации /server/database/init_db.sql для автоматического создания необходимых таблиц, связей и индексов.

 3. Запуск серверной части (Python)
1. Перейдите в папку /server.
2. Установите все необходимые зависимости:
   bash
   pip install -r requirements.txt
